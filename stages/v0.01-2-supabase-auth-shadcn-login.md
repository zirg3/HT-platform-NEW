## Добавлено

- **shadcn/ui**: `components.json`, `src/lib/utils.ts`, компоненты `button`, `input`, `label`, `card`; обновлён `globals.css`
- **Supabase SSR**: `src/lib/supabase/{client,server,middleware}.ts`, `src/middleware.ts`
- **Auth**: `src/app/login` (страница + server actions), `src/app/auth/callback`, `src/lib/auth/{paths,session}.ts`
- **Роли**: заглушки ЛК `/student`, `/teacher`, `/manager`, `/admin` с `requireRole` в layout
- **UI**: `LoginForm`, `AppShell`, `RolePlaceholder`
- **БД**: `supabase/migrations/20260517120000_initial_schema.sql` (profiles, courses, lessons, RLS, триггер профиля)
- **Типы**: `src/types/database.ts`
- **`.env.example`**: anon URL/key (+ комментарий service role)

## Изменено

- `src/app/page.tsx` — редирект авторизованных в кабинет по роли; гостям — кнопка «Войти»
- `package.json` — зависимости Supabase, shadcn, radix

## Как проверить

1. Применить миграцию в Supabase (см. `supabase/README.md`)
2. Создать пользователя с `app_metadata.role: "admin"`
3. `cp .env.example .env.local` → заполнить ключи
4. `npm run dev` → `/login`
