# v0.03-1 · Привязки, пользователи, ДЗ

**Ветка (план):** `dev-0.3` от `dev`

## Добавлено

- `lib/supabase/admin.ts` — service role для Admin API
- `lib/users/*` — типы и запросы профилей, привязок
- `app/actions/users.ts` — создание, правка, удаление пользователей (админ)
- `app/actions/assignments.ts` — привязка ученик ↔ преподаватель (staff)
- `completeLessonAction` — статус `completed` + upsert `homework`
- UI: `AdminNav`, `AssignmentsManager`, `UsersManager`
- Страницы: `/admin/users`, `/admin/assignments`, `/manager/assignments`, `/teacher/students`
- В диалоге урока: проведение + текст ДЗ, просмотр ДЗ у проведённых

## Поведение

- Менеджер и админ назначают привязки; преподаватель видит только своих учеников
- Админ создаёт пользователей (email, пароль, роль) через Supabase Auth Admin API
- Преподаватель отмечает урок проведённым и задаёт ДЗ (plain text)

## Проверка

1. `SUPABASE_SERVICE_ROLE_KEY` в `.env.local`
2. Админ: `/admin/users` — создать ученика и преподавателя
3. `/admin/assignments` — привязать ученика к преподавателю
4. Преподаватель: `/teacher` — урок → «Отметить проведённым» + ДЗ
5. Ученик: открыть проведённый урок — виден текст ДЗ
