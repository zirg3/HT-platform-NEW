# dev-0.01 · v0.01-1 — Основа приложения (Next.js + Tailwind + типы)

**Ветка (план):** `dev` под тегами `dev-0.01` … (переключение ветки — только по явной договорённости из `stages-rules.mdc`)

## Что сделано

- Сгенерировано приложение Next.js (**App Router**), **TypeScript**, **Tailwind CSS 4**, **ESLint**; сборка продакшен (`next build`) проходит.
- Приложение размещено в **`apps/web`**: корень монорепы/рабочего каталога с кириллицей в имени недопустим как `npm` `name`, поэтому каркас вынесен в подпапку с латиницей.
- Подключён **Zustand** (далее — локальное UI/клиентское состояние).
- Корневой **`layout`**: `lang="ru"`, русские метаданные сайта через константу; шрифты Geist — подмножества `latin` + `cyrillic`.
- Константы: **`SITE_NAME`**, **`DEFAULT_TIME_ZONE`** = `Europe/Moscow` (`src/lib/constants.ts`) под требование хранить время в UTC, отображение в TZ организации.
- Тип ролей из спеки: **`UserRole`** (`src/types/roles.ts`).
- **` .env.example`**: переменные для будущего Supabase (`NEXT_PUBLIC_SUPABASE_*`), без секретов.
- Главная страница: нейтральная светлая заглотовка текста без обучающего шаблона Vercel/Next.js.

## Структура (важное)

```

├── src/app/           # App Router — layout.tsx, page.tsx, globals.css
```

## Эффект / зачем

- Есть воспроизводимый **каркас** под Auth, middleware, календарь и ЛК без переписывания стека.
- Спека и дорожная карта могут пополняться отметками **`+`** по мере готовности модулей.
- Префиксы **`dev-0.01`** согласованы с локальными правилами версионирования итераций.
